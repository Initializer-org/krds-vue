# KRDS Vue 컴포넌트 라이브러리

<div className="sb-container">
  <div className="sb-section-title">
    ## 개요

    **KRDS (Korean Digital Service Design System) Vue**는 대한민국 디지털 정부 서비스를 위한 Vue 3 기반 컴포넌트 라이브러리입니다.

    사용자 인터페이스의 가장 작은 단위로 과업에 상관없이 일관성 있게 사용되는 공통 요소에 대한 가이드를 제공하며,
    접근성과 사용성을 고려한 표준화된 디자인 시스템을 구현합니다.

  </div>

  <div className="sb-section-title">
    ## 디자인 철학

    ### 사용자 중심 접근
    KRDS는 개발자가 디자인 시스템의 컴포넌트를 쉽게 적용할 수 있도록 가이드를 제공합니다.
    디자인 구현을 직관적이고 접근 가능하게 만드는 데 중점을 둡니다.

    ### 핵심 원칙
    - **일관성**: 모든 정부 디지털 서비스에서 동일한 사용자 경험 제공
    - **접근성**: WCAG 2.1 AA 수준의 웹 접근성 준수
    - **효율성**: 개발 생산성 향상을 위한 재사용 가능한 컴포넌트
    - **표준화**: 정부 서비스에 특화된 디자인 가이드라인 적용
    - **유연성**: 다양한 프로젝트 요구사항에 맞춘 커스터마이징 지원

  </div>

  <div className="sb-section-title">
    ## 디자인 시스템 특징

    ### 디자인 토큰
    - Figma 등 디자인 툴에서 디자인 토큰 추출 지원
    - JSON 기반 토큰 관리로 쉬운 업데이트
    - CSS 변수로 자동 변환되어 런타임 커스터마이징 가능

    #### 디자인 토큰 수정
    디자인 토큰은 디자인 파일에서 수정한 뒤 코드에 적용하는 것이 원칙이지만, 디자인 파일에서 직접 토큰을 관리할 수 없는 경우 코드에서 토큰값을 수정할 수 있습니다.

    디자인 토큰 수정 시에는 원본 코드값을 직접 수정하지 않고 프로젝트의 CSS 또는 SCSS 파일에 필요한 토큰 값(CSS variable)을 재선언합니다.

    ```css
    /* 사용자 파일에 코드를 재선언한다 */
    :root {
      --krds-color-light-primary-50: #ff00aa;
    }
    ```

    #### 수정 전 유의 사항
    - **디자인 토큰 관련 내용 숙지**: 디자인 토큰을 수정하기 전 관련 내용에 대한 숙지가 필요합니다
    - **영향도 검토**: 디자인 토큰 수정 시 다른 토큰에 미치는 영향도를 면밀히 검토합니다
    - **토큰 목록 확인**: 디자인 토큰의 전체 목록을 확인하고 수정 대상 토큰을 미리 식별해 둡니다

    ### 타이포그래피
    - **Pretendard GOV** 웹폰트를 표준 서체로 채택
    - 정부 서비스에 최적화된 가독성 제공
    - 다양한 굵기(Regular, Medium, Bold) 지원

    ### 유연한 구현
    - **NPM**, **GitHub**, **CDN** 등 다양한 설치 방법 제공
    - SCSS와 컴파일된 CSS 모두 제공
    - 전체 또는 부분 라이브러리 import 지원
    - 모듈식 컴포넌트 아키텍처

  </div>

  <div className="sb-section-title">
    ## 사용법

    ### 설치
    ```bash
    npm install @krds.ui/vue
    # or
    pnpm add @krds.ui/vue
    # or
    yarn add @krds.ui/vue
    ```

    ### 전체 컴포넌트 import
    ```typescript
    import { createApp } from 'vue'
    import KrdsVue from '@krds.ui/vue'
    import '@krds.ui/vue/dist/style.css'

    const app = createApp(App)
    app.use(KrdsVue)
    ```

    ### 개별 컴포넌트 import
    ```typescript
    import { KrdsButton, KrdsInput } from '@krds.ui/vue'
    ```

  </div>

  <div className="sb-section-title">
    ## 스토리북에서 테마 테스트하기

    이 스토리북에서는 KRDS 컴포넌트의 다양한 테마를 실시간으로 테스트할 수 있습니다.

    ### 테마 전환 방법
    1. **툴바 테마 버튼**: 상단 툴바에서 **페인트브러시 아이콘**을 클릭하세요
    2. **테마 선택**: 다음 중 원하는 테마를 선택할 수 있습니다:
       - **Light ☀️**: 기본 라이트 모드
       - **Dark 🌙**: 고대비 다크 모드
       - **Auto (System) 🖥️**: 시스템 설정에 따라 자동 전환

    ### 테마 적용 확인
    - 모든 컴포넌트 스토리에서 선택한 테마가 즉시 반영됩니다
    - `color-mode` mixin을 사용하는 컴포넌트들의 색상 변화를 확인할 수 있습니다
    - 접근성과 가독성이 각 테마에서 어떻게 보장되는지 테스트해보세요

    ### 개발자를 위한 참고사항
    컴포넌트에서 테마를 지원하려면 SCSS에서 `color-mode` mixin을 사용하세요:

    ```scss
    .my-component {
      @include color-mode(light) {
        background-color: var(--krds-light-color-bg-primary);
      }

      @include color-mode(high-contrast) {
        background-color: var(--krds-dark-color-bg-primary);
      }
    }
    ```
